This is the complete instructions for kinova gen 3 lite pick and place in gazebo and ros2 project.

This will be a complete ROS2 jazzy and kinova gen3 lite project spawned in gazzebo
It will  have only three packages

1. kinova_description

2. kinova_control

3. kinova_perception

The broader instruction for this project is to make the codebase simple interpretable well documented and easier to understand and maintain.

First we have to setup the workspace:
the workspace will be consisting of kinova gen3 lite robotic arm(fetch the original  URDF, mesh , stl, etc files from the actual github repositoriess)
then a table on top of which the robotic arm will be mounted
then there should be some cubes of colour( red, green , yello) these should be placed in the table without any containers
and there should be some some containers for each coloured cubes but those colour of these placing containers for each cube should be different than the cube colours
make sure the placing containers are well defined have calculated geometries so they dont overlap with each other and they must remain within the robotic workspace

Then the perception part:
The goal of this project is to identify the colour of each blocks and place them in correct  placing containers 
for this use only two rgb cameras by using plugins inside gazebo, 
one camera should be placed at the overhead on top at a higher distance so the robotic arm does not interfere with it and it could see all the workspace(xy plane) at once.

another camera should be placed at the side so it could monitor all of workspace z axis at once. 

make sure these camera feeds are actively processed for realtime visual data processing and it correctly identify the cube colours as well as the cube coordinates and the end effector coordinates.

they should publish the data to the topics camera_horizontal and camera_vertical and camera_horizontal_annotated and camera_vertical_annotated

then the control part:
make sure the control package is well set all joints are properly configured for actuation, for inverse kinematics use pinocchio(pip install pin)
make sure to first calculate some standardized waypoints that are to be used for efficient motion
like there should be
   start : this is the starting point of the robotic arm , it should be directly above the unsorted cubes , from this the robotic arm goes to the cubes and grabs them
   place_coloured : each placing bins should have dfferent different waypoints and these should be directly above the middle  point of placing bins so that robotic arm can just drop the cubes on top of these containers
   intermediate points : there should be few intermediate points for standardized motions

 Cube Grasping : Make sure that there should be a robust check for whether end effector(ee) has grasped the cube or not ; if the ee has not grasped the cube then it should not proceed to place the cubes in each coloured containers
 Intermediate Falling : make sure that if the cube falls midway then ee can pick it again and pllace it in correct containers
 No Shortcuts : make sure then no shortcut technique to be used , the ee should actually hold the cubes grap it pick and and pace it in the final containers.
